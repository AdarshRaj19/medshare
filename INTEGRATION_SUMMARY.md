## 🔄 DATA FLOW EXAMPLES

### Example 1: Donor Adds Medicine

```
STEP 1: FRONTEND
  User fills form: add_medicine.html
  - Medicine name: "Paracetamol"
  - Quantity: 10
  - Expiry: 2026-12-31
  - Image: medicine.jpg
  
STEP 2: SEND TO BACKEND
  POST /add-medicine/
  Content-Type: multipart/form-data
  
STEP 3: BACKEND VALIDATION
  add_medicine() view:
  - Check user authenticated ✓
  - Check user is donor ✓
  - Validate form with MedicineForm
  
STEP 4: FORM VALIDATION
  MedicineForm.clean():
  - Validate expiry_date > today ✓
  - Validate quantity > 0 ✓
  - Validate image format ✓
  
STEP 5: SAVE TO DATABASE
  medicine = Medicine.objects.create(
    donor=request.user,
    name="Paracetamol",
    quantity=10,
    expiry_date="2026-12-31",
    status="available"
  )
  
  Image uploaded to: media/medicines/paracetamol_12345.jpg
  
STEP 6: RESPONSE TO FRONTEND
  Redirect to: /donor/dashboard/
  Message: "Medicine added successfully"
  
STEP 7: FRONTEND DISPLAY
  donor_dashboard.html:
  - Query database for user's medicines
  - Display in table with new entry
  - Show success message
```

### Example 2: NGO Requests Medicine

```
STEP 1: FRONTEND
  NGO browses medicines via search.html
  Clicks "Request Medicine"
  
STEP 2: BACKEND SEARCH
  search_medicines() view:
  Query database:
  SELECT * FROM medicine
  WHERE status='available'
  AND category = ?
  
STEP 3: DISPLAY RESULTS
  medicine_detail.html shows:
  - Medicine info from database
  - Donor contact info
  - Request button
  
STEP 4: SUBMIT REQUEST
  POST /medicine/<id>/request/
  Quantity: 5
  
STEP 5: BACKEND PROCESSES
  request_medicine() view:
  - Create DonationRequest record
  - Save to database
  - Send notification
  
  INSERT INTO donation_request
  (medicine, ngo, quantity, status)
  VALUES (123, 45, 5, 'pending')
  
STEP 6: RESPONSE
  Redirect to: /ngo/dashboard/
  Show pending requests
  
STEP 7: ADMIN NOTIFIED
  Admin sees request in admin panel
  Can approve/reject
  
STEP 8: DELIVERY FLOW
  If approved:
  - Assign delivery boy
  - Create delivery record
  - Send notifications
```

### Example 3: Real-time Delivery Tracking

```
STEP 1: DELIVERY BOY APP
  JavaScript captures GPS:
  navigator.geolocation.getCurrentPosition()
  latitude: 28.7041
  longitude: 77.1025
  
STEP 2: SEND TO BACKEND API
  AJAX POST to /api/delivery/123/update-location/
  {
    "latitude": 28.7041,
    "longitude": 77.1025
  }
  
STEP 3: BACKEND API VERIFICATION
  update_location() view:
  - Verify delivery boy authenticated
  - Validate coordinates
  - Check delivery exists
  
STEP 4: SAVE TO DATABASE
  location = DeliveryLocation.objects.create(
    delivery_id=123,
    latitude=28.7041,
    longitude=77.1025,
    timestamp=now()
  )
  
STEP 5: RETURN JSON RESPONSE
  {
    "status": "success",
    "message": "Location updated",
    "timestamp": "2026-01-31 17:30:00"
  }
  
STEP 6: NGO APP FETCHES LOCATIONS
  GET /api/delivery/123/locations/
  
STEP 7: BACKEND RETURNS LOCATION HISTORY
  [
    { lat: 28.7000, lon: 77.1000, time: "17:00" },
    { lat: 28.7020, lon: 77.1010, time: "17:10" },
    { lat: 28.7041, lon: 77.1025, time: "17:30" }
  ]
  
STEP 8: NGO UPDATES MAP
  JavaScript:
  - Plot delivery boy location
  - Show movement trail
  - Update in real-time
  - Show ETA
```
